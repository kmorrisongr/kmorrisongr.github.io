sara is a simple and lightweight dynamic window manager. What started out as an attempt to make catwm into a small and fully-functional window manager has turned into my primary project in C. At the moment, sara combines aspects of dwm and bspwm with some custom features. As my needs/wants change, so too will the program, and probably how much you like it.

If things like dwm-ipc scare the hell out of you because they're so big, this window manager might be for you. You want a window manager that works like a souped-up dwm with sane defaults that is also opinionated like all the cool kids. I think the combination of the tag system, the flexibility of sxhkd and IPC interaction, and the ease of parsing the window manager's internal state are good selling points.

I manage a Minecraft server that I run from a Raspberry Pi 4 (2GB) using Paper. Before I learned that Minecraft Java Edition isn't multithreaded, I set up a small Kubernetes (k3s) cluster to parallelize this, gaining some experience with k3sup and helm.

peachbar was a set of shell scripts to interface with lemonbar, a program that displays text from standard output in a status bar window.

In its first incarnation as sbar, it was a collection of separate .sh files that atomically (using mkdir) wrote information to a file that contains the pre-formatted string to be written to the X server's root window name. This string was then displayed using sara's internal status bar, Ã  la dwm.

As my interest in Unix philosophy grew, I started reducing the complexity of sara, which had become my daily-driver window manager. This included removing the built-in status bar, which facilitated a migration to lemonbar. The next iteration of the bar scripts then became sbar-lemon, which wrote text to a FIFO, which was then aggregated and sent to lemonbar.

However, the setup lacked flexibility. My final version, which became peachbar, was aimed at extreme modularity and asynchronous performance. My goal was to use only shell scripts. After much work, however, it became apparent that shell is not the ideal tool for some tasks, and I became intimately familiar with Not-Invented-Here syndrome. Once I discovered that the very powerful and flexible polybar supports lemonbar-formatted strings, I realized I had a quick way to integrate my custom window manager with a pre-existing, very useful tool, and jumped ship.
